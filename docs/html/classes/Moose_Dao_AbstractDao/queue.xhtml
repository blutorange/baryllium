Moose\Dao\AbstractDao::queue

Puts an entity into the queue. You can call {@link AbstractDao::persistQueue} to validate and perists all entities in the queue later. Note that Doctrine caches entities internally as well, you need to call {@link EntityManager::flush} to write all changes to the database. This method may be useful when you do not want to check the result of {@link AbstractDao::persist} all the time for validation errors.
